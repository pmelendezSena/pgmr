pineappledreams Apr 6, 2019

rails 5.2.3

Any insight would be most appreciated.

@total-anachronism , I got some good news for you, bud. I am running the same specs as you, windows 10 and everything.

I looked into this Japanese blogpost: https://qiita.com/shita_fontaine/items/aa3f762d7afa20c5d42a and man, I finally got it to work. Here's how.

First when creating a new rails project, make sure to add --skip-bundle into the end. So, for example, if you are following getting started section on railsguides and wanna create a blog the command on your shell command should be:

$ rails new blog --skip-bundle

This will allow you to get a jump on the damn thing. If you let the bundler run in the end and then try to change the gemfile, bundler will stubbornly install 1.4.0 version of sqlite3, which will fail to compile every single time.

So once it generates the base for you and you have successfully skipped the bundler in the end, NOW
change the line in the GemFile:

# Use sqlite3 as the database for Active Record
gem 'sqlite3'

to (as @WilfredoAlarcon has shown)

# Use sqlite3 as the database for Active Record
gem 'sqlite3', git: "https://github.com/larskanis/sqlite3-ruby", branch: "add-gemspec"

then run on your shell

$ bundle install

You will see that bundler will say bundle complete!, install the desired 1.3.13 version of sqlite3 from the git repo and when you run rails server, and breathe in relief that you will be able to access the damn server you worked so hard to get on to.

Let me know how it goes - because I did some wicked dragging and dropping of sqlite files and setting paths and whatever and it just kinda worked at one point from now on. If it works for you too - then great!
